i-feel-dirty 1.4.5 – a ZenPhoto/ZenPage theme

Please report bugs on the forum and I will try to fix as soon as possible.

i-feel-dirty is a theme initialy created for Wordpress by Studio ST.

I found this theme by chance, and as I liked it, I decided to adapt it to ZenPhoto respecting as best as possible the graphics of the original.

Feel free to download and use it, and thanks in advance for your feedback!

Description

Important: To use the release 1.4.5 of the theme, you should have ZenPhoto 1.4.5 (or more).
If you use an older release of ZenPhoto, see Archives of i-feel-dirty.

The theme supports the following ZenPhoto plugins:

  • colorbox, comment_form, contact_form, dynamic-locale, favoritesHander, flag_thumbnail, GoogleMap, print_album_menu, rating, register_user, RSS (new),slideshow, user_login-out, zenpage

Additional scripts used:

  • script to navigate with arrow keys on news and images.

Improvements of 1.4.5 release

  • Switch to HTML5
  • Improvements to allow slideshows in Favorites (use improvements of zenphoto 1.4.5)
  • Improvements to be compliant with new contact, comment and register forms and with other ZenPhoto 1.4.5 new features (read changelog.txt)
  • Improvements to use RSS plugin
  • Some graphical improvements (read changelog.txt)
  • Fix some issues (read changelog.txt)

Installation

  1. Upload the zip file to your computer
  2. Unzip the downloaded zip file locally, and upload the i-feel-dirty folder to the directory /themes/ of your ZenPhoto site
  3. In ZenPhoto administration, go to the Themes tab and activate the i-feel-dirty theme
  4. Navigate to Options>Theme to view and configure the available options for i-feel-dirty

For the theme works well, I advise you:

  • To enter a gallery title and a website title in admin>options>gallery
  • If you use the user_login-out plugin, to check "User name" in admin>options>security

 

 

English version - Version Française

i-feel-dirty 1.4.5 – un thème ZenPhoto/ZenPage

Rapportez les bugs sur le forum ZenPhoto.org et j’essaierais de les corriger aussitôt que possible.

i-feel-dirty est un thème initialement créé pour Wordpress par Studio ST.

J'ai trouvé ce thème par hasard en fouillant sur Internet et comme je l'ai trouvé très sympa, j'ai décidé de l'adapter sur ZenPhoto en respectant le mieux possible la charte graphique de l'original.

N'hésitez pas à le télécharger et à l'utiliser, et merci d'avance pour vos avis et commentaires !

Description

Important : Pour utiliser la version 1.4.5 du theme, vous devriez utiliser ZenPhoto 1.4.5 (ou plus).
Si vous utilisez une ancienne version de ZenPhoto, consultez les Archives de i-feel-dirty.

Le thème supporte les plugins ZenPhoto suivants :

  • colorbox, comment_form, contact_form, dynamic-locale, favoritesHander, flag_thumbnail, GoogleMap, print_album_menu, rating, register_user, RSS (nouveau),slideshow, user_login-out, zenpage

Scripts additionnels utilisés :

  • script permettant la navigation avec les flèches du clavier sur les actualités et les images.

Améliorations de la version 1.4.5

  • Bascule vers HTML5
  • Améliorations pour gérer les diaporamas dans les Favoris (utilise les améliorations de ZenPhoto 1.4.5)
  • Améliorations pour être compatible avec les nouveaux formulaires de contact, commentaires et d'enregistrement et avec les autres améliorations de Zenphoto 1.4.5 (voir changelog.txt)
  • Améliorations pour gérer le plugin RSS
  • Quelques améliorations graphiques (voir changelog.txt)
  • Corrections de quelques bugs (voir changelog.txt)

Installation

  1. Téléchargez le fichier zip sur votre ordinateur
  2. Décompressez le fichier zip localement, et télécharger le dossier i-feel-dirty dans votre répertoire /themes/ de votre site ZenPhoto
  3. Dans l'administration ZenPhoto, allez sur l'onglet Thèmes et activer le thème i-feel-dirty
  4. Allez sur Options>Thème afin d'afficher et configurer les options disponibles pour i-feel-dirty

Pour que le thème fonctionne bien, je vous conseille :

  • d'entrer un titre de la galerie et un titre du site dans admin>options>galerie
  • si vous utilisez le plugin user_login-out, de cocher la case "Nom de l’utilisateur" dans admin>options>sécurité
